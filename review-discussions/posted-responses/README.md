# Posted Response Archive

**This folder contains copies of responses we've actually posted to the public JOSS review issue**

---

## 📂 **Purpose**

Keep a record of what we posted publicly, so we can:
- Track what we've already said
- Reference our previous responses
- Learn from what worked/didn't work
- Have backup if JOSS issue gets modified

---

## 📝 **Naming Convention**

Use this format: `YYYY-MM-DD-topic-brief-description.md`

**Examples:**
- `2025-10-20-installation-process.md`
- `2025-10-22-baseline-methodology-explanation.md`
- `2025-10-25-reviewer2-performance-claims.md`

---

## 📋 **What to Include**

Each archived response should have:

1. **Link to JOSS issue comment** (the actual public URL)
2. **Full text of what was posted**
3. **Which reviewer/comment it addressed**
4. **Date posted**
5. **Who posted it** (Takahashi or Shimizu)

---

## 🔄 **Workflow**

1. Draft response in `response-drafts/` folder
2. Get approval from both team members
3. Post to public JOSS issue
4. Copy the posted text here with metadata
5. Mark as posted in SIMPLE-RESPONSE-PLAN.md

---

**Files will appear here once you start posting responses to JOSS reviewers**
